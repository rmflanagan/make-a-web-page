Lesson 1: The Basics of the Web & HTML

Basics of the Web:

The world wide web is comprised of a large collection of HTML documents. These documents are connected to each other by hyper links. HTML documents may contain plain text, html, images, videos and/or music.

The main components of the web are you, the user, your computer with its web browser, the internet and servers. The user goes to a web page (e.g. http://www.google.com) and your browser sends an http request via the internet to servers; these servers respond to these requests by locating the appropriate page(s) and displaying it back on your web browser. This video contains a visual representation of these web component relationships.

HTML:

HTML is an abbreviation of hypertext markup language. HTML is comprised of text content, or what the user sees, markup, or how the user sees it, references to other documents, such as images or videos, and finally links to other pages. This video gives an HTML overview.

HTML Tags/Elements:

HTML markup is written as elements that describe how contents are displayed in a document. To write these elements, we use HTML tags. This video shows the structure of tags. Examples of basic tags are those used for displaying bold text (b for bold) and italic text (em for emphasis).

A "Smart" Aside about "Stupid" Computers:

Computers are "stupid" in the sense that they can only literally translate programming code. They cannot interpret what a programmer intended, and are therefore unable to correct mistakes programmers make. Even small errors in code can lead to large problems in the rendered results.

HTML Attributes:

Some HTML tags have attributes, which use an additional value in the opening tag. An example is the anchor tag, as described in this video, that uses the attribute "href" to turn text into a link to another page. A further example is the image tag, that contains the attribute value "scr" to display an image file.

Another Aside about Remembering:

It's important to note that even experienced programmers don't remember how to code everything, as there is simply too much to remember! If you forget how to do something, a simple online search will typically give you the answer you need.

Inline/Block Elements:

HTML elements are classified as either inline or block. Inline elements are just text, whereas block elements create an "invisible box" around text. This video uses the break and paragraph tags to describe the distinction between inline and block tags.

Container Elements:

Container elements simply "contain" whatever is found within them. The example given in this video is "span", an inline element, and "div", a block element.

HTML Document Structure:

HTML documents need doctype, html, head and body tags. Doctype has been simplified since the use of html5. The head section may contain things such as the document meta-data, CSS styling elements and the title of the page. The body section contains the actual text of the page. This video has a visual representation of the structure of HTML documents.

Lesson 2: Creating a Structured Document with HTML

Developer Tools:

Web browsers have developer tools built into them to help developers break down the structure of web pages and see how the browser interprets its components. Developer tools help you work with all the components of a page, such as HTML, CSS and Javascript.

The HTML components are the structure of the page; examples are the actual text and images on a page. The CSS components are the style of the page, or how that text and image is displayed; a basic example is the particular size font of text on the page. The Javascript components are the interactive things on the page. This video shows how developer tools are used to explore the many components of Wikipedia's main page.

HTML-CSS-DOM:

A browser knows how to turn an HTML tag into an element in a tree structure because of DOM, or Document Object Model. A browser knows how to format the HTML elements because of CSS, or Cascading Style Sheets. This video gives an overview of how the HTML, CSS and DOM of a particular page relate.

Translating Boxes into HTML:

Everything on a page is in a "box". Once a web page design is broken down into simple boxes, these boxes can be represented as HTML elements and formatted using CSS. This video shows how boxes are translated into HTML using div tags, and then each div is given logical class attributes.

This code is typically written in a text editor designed specifically for programming, such as Scratchpad, Codepen or Sublime Text. The programmer decides which text editor to utilize based on the type and complexity of the page.

Lesson 3: Adding CSS Style & HTML Structure

Avoid Repetition:

Avoiding repetition in programming is very important, as it helps establish consistency. The more times you make a mistake, the more times that mistake would need to be corrected. If repetition is avoided, it minimizes the need for repeated corrections.

Similar elements on a page should be grouped in classes so that styles can be applied to all the elements of the same type. Class names should be logical and easy to discern.

CSS Introduction:

Cascading Style Sheets, or CSS, is a programming language used to apply formatting to HTML documents. "Cascading" means that the most specific rule is applied to that element, and related elements depending upon the tree-like structure (DOM) of the HTML document.This video uses an analogy to national, state, municipal and house laws to illustrate the concept of "cascading".

There are three ways to apply CSS to an HTML document. The first is to write the CSS in the head element of the HTML document. The second, and most efficient, is to reference a separate CSS file in the head element of the HTML document. The third is to include the CSS inline with the HTML; this is not recommended as it leads to a lot of repeated code.

CSS Details:

CSS uses selectors to control the style of HTML elements. The selectors tell the browser to apply a style (such as font type, color, etc.) to all elements of the selected type. Elements can also be selected by class so as to apply a style across all elements in that particular class.

Whenever possible, programmers should use semantic tags to define elements. Semantic tags more clearly tell the browser and the programmer what the element actually contains. For example, a "div" tag doesn't tell the browser that the page's header text is inside, whereas a "header" tag does.

Anatomy of a Box:

Boxes have a specific structure, and knowing that structure allows for the modification of any or all of its parts. The content is surrounded by padding, border and margin areas, in that respective order. CSS is used to control any particulars about these box components, such as width, color, etc. This video reviews all box components and how they can be manipulated using CSS.

Getting boxes sized correctly across different screen sizes and devices can be challenging. Setting box widths as a percentage (rather than a set amount of pixels) of screen size and using flex boxes are two of the most important ways programmers can gain flexibility and accuracy in box design.

Document Design Summary & Review:

The steps of transforming a design mock-up into an HTML document are:

  1. Look for natural boxes.
  2. Look for repeated styles and semantic elements.
  3. Write the HTML.
  4. Apply styles using CSS.
  5. Use Code-Test-Refine to fix things, starting with the largest details and finishing with the smallest details. Code-Test-Refine is critical in the programming process as it is where errors are found and corrected in code used across multiple browsers and devices.

All documents should have their HTML and CSS checked by an outside validator to be sure the page displays correctly across browsers. The World Wide Web Consortium (W3C) provides both an HTML and a CSS validator.
